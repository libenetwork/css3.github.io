Підключення CSS3

Для того щоб використовувати CSS3 достатньо створити файл з розширенням СSS і підклюити до HTML-файла. Або вказати необхідні властивості у сам документи. Властивості CSS можна використовувати трьома способами

1

Прописування властивостей в тег

За допомогою атрибуту style властивість можна надати будь-якому HTML-тегу

<div style = "color:blue">
   <p style = "font-family: Roboto"></p>
</div>

Копіювати

Переваги:

  • Можливість налаштувати кожен окремий елемент
  • Не потрібне створення окремого файлу

Недоліки:

  • Громіздкість. Код складніше читається
  • Необхідність дублювати властивості для схожих об'єктів, що знижує швидкість роботи

2

Прописування властивостей в тег style

Окрім атрибуту style існує також однойменний тег. Його можна використовувати всередині HTML-документа

<head>
  <style>
    body{
      background-color: black;
    }
    .hello{
       font-family: Montserrat;
       color: white;
    }
  </style>
</head>
<body>
  <p class = "hello">Hello World!</p>
</body>

Копіювати

Переваги:

  • Не потрібно створювати додатковий файл CSS
  • Можна створити властивість актуальну тільки для однієї сторінки, аби неускладнювати основний CSS код

Недоліки:

  • Властивості будуть актуальні тільки для цієї сторінки. Використовувати її на інших сторінках неможливо.

3

Підключення CSS документів до сторінки

Для того щоб скористатися даним методом необхідно на сервері додати файл із розширенням CSS

<head>
  <link rel = "stylesheet" href = "styles.css">
</head>

Копіювати

Переваги:

  • Спрощує читання HTML-документу
  • Дозволяє використовувати одні й ті самі властивості на декількох сторінках

Недоліки:

  • Необхідно створювати додатковий CSS-файл
  • Через свою універсальність, іноді це знижує гнучкість налаштування

Метод яким варто підключати CSS необхідно обирати виходячи із кожної окремої задачі, намагаючись робити код легким і простим для читання

Фреймворки

Фреймворки це вже готові набори CSS файлів, направлені на спрощення роботи. Фреймворки включають в себе стилі, що часто використовуються, або спрощують видозміну тих чи інших стилів.

Bootstrap

  • Широкий набір інструментів для роботи із адаптивністю
  • Елементи інтерфейсу
  • Хороша документованість

Tailwind

  • Висока гнучкість у кастомізації елементів
  • Власні шаблони для створення інтерфейсу
  • Широке ком'юніті і група підтримки

Material

  • Якісна документація
  • Підтримка з боку Google
  • Набір готових дизайнерський рішень

Foundation

  • Величезний інструментарій
  • Гнучкість
  • Великий набір елементів інтерфейсів
